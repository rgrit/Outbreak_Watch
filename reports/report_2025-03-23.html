CVE-2025-30472

Description:
Corosync through 3.1.9, if encryption is disabled or the attacker knows the encryption key, has a stack-based buffer overflow in orf_token_endian_convert in exec/totemsrp.c via a large UDP packet.
Summary:
This vulnerability in Corosync through version 3.1.9 can be exploited through a stack-based buffer overflow in the encryption process or if the attacker has the encryption key. This could be done via a large UDP packet. This vulnerability could affect anyone using Corosync without encryption or with a known encryption key. To mitigate this risk, users should update to a version of Corosync that addresses this issue and ensure encryption is enabled with secure encryption keys.

CVE-2024-13739

Description:
The Newsletters plugin for WordPress is vulnerable to Reflected Cross-Site Scripting via the "to" parameter in all versions up to, and including, 4.9.9.7 due to insufficient input sanitization and output escaping. This makes it possible for unauthenticated attackers to inject arbitrary web scripts in pages that execute if they can successfully trick an admin user into performing an action such as clicking on a link.
Summary:
There is a security vulnerability in the Newsletters plugin for WordPress that allows attackers to inject harmful scripts onto web pages if they can trick an admin user into clicking on a link. This affects all versions up to 4.9.9.7. To protect against this threat, users should update to the latest version of the plugin and be cautious when clicking on unfamiliar links.

CVE-2025-0723

Description:
The ProfileGrid ‚Äì User Profiles, Groups and Communities plugin for WordPress is vulnerable to blind and time-based SQL Injections via the rid and search parameters in all versions up to, and including, 5.9.4.7 due to insufficient escaping on the user supplied parameter and lack of sufficient preparation on the existing SQL query. This makes it possible for authenticated attackers, with Subscriber-level access and above, to append additional SQL queries into already existing queries that can be used to extract sensitive information from the database.
Summary:
The ProfileGrid WordPress plugin is at risk of SQL Injections, which can allow attackers with certain access levels to access sensitive information from the database. Users of this plugin should update to version 5.9.4.8 or higher to patch this vulnerability and protect their data.

CVE-2025-0724

Description:
The ProfileGrid ‚Äì User Profiles, Groups and Communities plugin for WordPress is vulnerable to PHP Object Injection in all versions up to, and including, 5.9.4.5 via deserialization of untrusted input in the get_user_meta_fields_html function. This makes it possible for authenticated attackers, with Subscriber-level access and above, to inject a PHP Object. If a POP chain is present via an additional plugin or theme installed on the target system, it may allow the attacker to perform actions like delete arbitrary files, retrieve sensitive data, or execute code depending on the POP chain present.
Summary:
The ProfileGrid plugin for WordPress is vulnerable to a security issue that could allow attackers with certain levels of access to inject malicious code into a website. This vulnerability only becomes a threat if the website is also using another plugin or theme that contains a specific type of exploit chain. Website administrators should update the ProfileGrid plugin to the latest version to protect against this vulnerability.
